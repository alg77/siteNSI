01 — Introduction

Qu'est-ce qu'un arbre ?

Un arbre est une structure de données hiérarchique composée de nœuds reliés entre eux par des arêtes. C'est l'une des structures fondamentales de l'informatique, omniprésente dans les algorithmes, les langages et les systèmes.

📐 Définition formelle
Un arbre est un graphe connexe sans cycle, orienté depuis un nœud particulier appelé racine. Chaque nœud possède exactement un parent (sauf la racine) et peut avoir zéro ou plusieurs enfants.

Pourquoi les arbres sont-ils importants ?

Les arbres sont utilisés partout en informatique :

  • Systèmes de fichiers : l'arborescence de dossiers de votre OS est un arbre
  • Compilateurs : l'arbre syntaxique abstrait (AST) représente le code source
  • Bases de données : les index B-trees accélèrent les requêtes
  • Intelligence artificielle : arbres de décision, arbre minimax dans les jeux
  • HTML/XML : le DOM (Document Object Model) est un arbre
  • Tri et recherche : les arbres binaires de recherche (ABR)

Lien avec les autres notions NSI

  • Récursivité : la quasi-totalité des algorithmes sur les arbres sont récursifs
  • Listes chaînées : un arbre généralise la liste (chaque nœud peut avoir plusieurs successeurs)
  • Graphes : un arbre est un graphe acyclique connexe orienté
  • Programmation orientée objet : les arbres se représentent naturellement avec des classes
💡 À retenir pour le bac
Au baccalauréat NSI, les arbres apparaissent régulièrement. On vous demande souvent d'écrire des fonctions récursives (hauteur, taille, parcours) et de dessiner ou lire un arbre à partir d'une représentation Python.

⚠ Confusion fréquente
Hauteur vs Profondeur : la profondeur d'un nœud est mesurée depuis la racine (vers le bas). La hauteur d'un nœud est mesurée vers les feuilles (vers le bas aussi, mais à partir du nœud lui-même). La hauteur de l'arbre = hauteur de sa racine = profondeur maximale.

Arbres binaires : propriétés importantes

  • Un arbre binaire de hauteur h contient au maximum 2h+1 − 1 nœuds
  • Un arbre binaire de hauteur h contient au moins h + 1 nœuds
  • Un arbre binaire complet a toutes ses feuilles au même niveau
  • Un arbre binaire parfait est complet et tous les nœuds internes ont exactement 2 enfants
03 — Représentation

Représentation des arbres

Représentation graphique

Un arbre binaire se dessine avec la racine en haut, les fils gauche (FG) à gauche et les fils droits (FD) à droite :

         8           ← racine
        / \
       3   10
      / \    \
     1   6   14
        / \   /
       4   7 13

Cet arbre est un arbre binaire de recherche (ABR) : pour chaque nœud, les valeurs à gauche sont inférieures et celles à droite sont supérieures.

Représentation en Python — classe Noeud

En NSI, on représente un arbre binaire avec une classe Noeud :

arbre.py — Classe Noeud
class Noeud:
    """Représente un nœud d'un arbre binaire."""

    def __init__(self, valeur, gauche=None, droite=None):
        self.valeur = valeur        # donnée stockée dans le nœud
        self.gauche = gauche        # sous-arbre gauche (Noeud ou None)
        self.droite = droite        # sous-arbre droit  (Noeud ou None)

    def est_feuille(self):
        """Retourne True si le nœud est une feuille."""
        return self.gauche is None and self.droite is None

    def __repr__(self):
        return f"Noeud({self.valeur})"

Construction d'un arbre en Python

Construction manuelle de l'arbre
# Construction de l'arbre :

04 — Parcours

Parcours d'arbres

Parcourir un arbre signifie visiter tous ses nœuds exactement une fois. Il existe plusieurs stratégies, chacune avec des applications différentes.

Pour tous les exemples, on utilise cet arbre de référence :

         A
        / \
       B   C
      / \   / \
     D   E F   G

Parcours en profondeur (DFS — Depth-First Search)

Les parcours en profondeur explorent un sous-arbre entièrement avant de passer au suivant. Ils se définissent par l'ordre de visite du nœud courant (N) par rapport à ses fils gauche (G) et droit (D).

Parcours préfixe (N → G → D)

On visite d'abord le nœud, puis le sous-arbre gauche, puis le sous-arbre droit.

Résultat sur l'exemple : A → B → D → E → C → F → G

Parcours préfixe — Python
def prefixe(noeud):
    """Parcours préfixe (NoeudGaucheDroit) — récursif."""
    if noeud is None:     # cas de base : arbre vide
        return
    print(noeud.valeur)   # 1. Visiter le nœud
    prefixe(noeud.gauche) # 2. Parcourir le sous-arbre gauche
    prefixe(noeud.droite) # 3. Parcourir le sous-arbre droit

Parcours infixe (G → N → D)

On parcourt le sous-arbre gauche, puis on visite le nœud, puis le sous-arbre droit.

Résultat sur l'exemple : D → B → E → A → F → C → G

Propriété clé : sur un arbre binaire de recherche, le parcours infixe donne les valeurs dans l'ordre croissant !

Parcours infixe — Python
def infixe(noeud):
    """Parcours infixe (GauchNoeudDroit) — récursif."""
    if noeud is None:
        return
    infixe(noeud.gauche)  # 1. Parcourir le sous-arbre gauche
    print(noeud.valeur)    # 2. Visiter le nœud
    infixe(noeud.droite)  # 3. Parcourir le sous-arbre droit

Parcours postfixe (G → D → N)

On parcourt d'abord les deux sous-arbres, puis on visite le nœud en dernier.

Résultat sur l'exemple : D → E → B → F → G → C → A

Application : utile pour supprimer un arbre (on supprime les enfants avant les parents) ou évaluer une expression arithmétique en notation polonaise inversée.

Parcours postfixe — Python
def postfixe(noeud):
    """Parcours postfixe (GaucheDroitNoeud) — récursif."""
    if noeud is None:
        return
    postfixe(noeud.gauche) # 1. Parcourir le sous-arbre gauche
    postfixe(noeud.droite) # 2. Parcourir le sous-arbre droit
    print(noeud.valeur)     # 3. Visiter le nœud

Parcours en largeur (BFS — Breadth-First Search)

On visite les nœuds niveau par niveau, de gauche à droite.

Résultat sur l'exemple : A → B → C → D → E → F → G

Ce parcours n'est pas récursif : il utilise une file (deque).

Parcours en largeur — Python (avec file)
from collections import deque

def largeur(racine):
    """Parcours en largeur (BFS) — utilise une file."""
    if racine is None:
        return

    file = deque([racine])  # on enfile la racine

    while file:
        noeud = file.popleft()       # on défile
        print(noeud.valeur)           # on visite
        if noeud.gauche is not None:
            file.append(noeud.gauche) # on enfile le fils gauche
        if noeud.droite is not None:
            file.append(noeud.droite) # on enfile le fils droit

Tableau récapitulatif des parcours

ParcoursOrdreRésultat (arbre A)Application typique

InfixeG → N → DD B E A F C GTri sur ABR (ordre croissant)

🔑 Astuce mémorisation
Le nom du parcours indique la position du Nœud dans l'ordre N-G-D :
• Préfixe = Nœud en premier (avant)
• Infixe = Nœud au milieu (entre)
• Postfixe = Nœud en dernier (après)
05 — Opérations

Opérations sur les arbres

Calcul de la taille

La taille d'un arbre est le nombre total de nœuds. On la calcule récursivement : taille = 1 (nœud courant) + taille(gauche) + taille(droite).

Taille d'un arbre
def taille(noeud):
    """Retourne le nombre de nœuds de l'arbre."""
    if noeud is None:       # arbre vide → taille 0
        return 0
    return 1 + taille(noeud.gauche) + taille(noeud.droite)
    # 1 pour le nœud courant, + récursion sur les deux sous-arbres

Calcul de la hauteur

La hauteur d'un arbre est le nombre d'arêtes sur le plus long chemin de la racine à une feuille. Conventionnellement, la hauteur d'un arbre vide est −1, celle d'un arbre réduit à une feuille est 0.

Hauteur d'un arbre
def hauteur(noeud):
    """Retourne la hauteur de l'arbre (-1 si vide, 0 si feuille)."""
    if noeud is None:
        return -1              # arbre vide : convention hauteur = -1
    h_gauche = hauteur(noeud.gauche)
    h_droite = hauteur(noeud.droite)
    return 1 + max(h_gauche, h_droite)
    # +1 pour l'arête vers ce nœud, max pour prendre le chemin le plus long
⚠ Convention hauteur arbre vide
Certains sujets définissent la hauteur d'un arbre vide à 0 et celle d'une feuille à 1. Lisez toujours attentivement l'énoncé. La logique reste la même, seule la constante de base change.

Recherche dans un arbre binaire quelconque

Recherche dans un arbre binaire
def recherche(noeud, valeur):
    """Cherche une valeur dans un arbre binaire quelconque.
    Retourne True si trouvée, False sinon."""
    if noeud is None:
        return False             # arbre vide → non trouvé
    if noeud.valeur == valeur:
        return True              # trouvé !
    # Chercher dans les deux sous-arbres
    return recherche(noeud.gauche, valeur) or \
           recherche(noeud.droite, valeur)

Recherche dans un Arbre Binaire de Recherche (ABR)

Dans un ABR, la recherche est beaucoup plus efficace car on peut éliminer la moitié de l'arbre à chaque étape :

Recherche dans un ABR (optimisée)
def recherche_abr(noeud, valeur):
    """Recherche dans un ABR — O(log n) en moyenne."""
    if noeud is None:
        return False             # non trouvé
    if valeur == noeud.valeur:
        return True              # trouvé
    elif valeur < noeud.valeur:
        return recherche_abr(noeud.gauche, valeur)  # aller à gauche
    else:
        return recherche_abr(noeud.droite, valeur) # aller à droite

Insertion dans un ABR

Insertion dans un ABR
def insertion_abr(noeud, valeur):
    """Insère une valeur dans un ABR. Retourne la nouvelle racine."""
    if noeud is None:
        return Noeud(valeur)   # créer un nouveau nœud feuille
    if valeur < noeud.valeur:
        noeud.gauche = insertion_abr(noeud.gauche, valeur)
    elif valeur > noeud.valeur:
        noeud.droite = insertion_abr(noeud.droite, valeur)
    # si valeur == noeud.valeur : doublon ignoré
    return noeud               # retourner le nœud (éventuellement modifié)

# Exemple d'utilisation :
racine = None
for val in [8, 3, 10, 1, 6, 14]:
    racine = insertion_abr(racine, val)
# Résultat : ABR avec racine 8

Compter les feuilles

Nombre de feuilles
def nb_feuilles(noeud):
    """Retourne le nombre de feuilles de l'arbre."""
    if noeud is None:
        return 0
    if noeud.est_feuille():
        return 1               # c'est une feuille
    return nb_feuilles(noeud.gauche) + nb_feuilles(noeud.droite)
06 — Complexité

Complexité algorithmique

Complexité des parcours

Tous les parcours (préfixe, infixe, postfixe, largeur) visitent chaque nœud exactement une fois. Leur complexité est donc :

O(n) Parcours préfixe O(n) Parcours infixe O(n) Parcours postfixe O(n) Parcours en largeur

où n est le nombre de nœuds. On ne peut pas faire mieux : pour visiter n nœuds, il faut au minimum n opérations.

Complexité des opérations sur un ABR

OpérationCas moyenPire cas (arbre dégénéré)
RechercheO(log n)O(n)
InsertionO(log n)O(n)
TailleO(n) (toujours)
HauteurO(n) (toujours)

Cas favorable vs défavorable

🌳 Arbre équilibré — Cas favorable
Un arbre équilibré (toutes les feuilles à peu près à la même profondeur) de n nœuds a une hauteur de l'ordre de log₂(n). La recherche dans un ABR équilibré est donc en O(log n), comparable à la dichotomie. 
  Arbre équilibré (h ≈ log n)        Arbre dégénéré (h = n-1)
         5                                  1
        / \                                  \
       3   7                                 2
      / \ / \                                 \
     2 4 6 8                                3
  hauteur = 2                                  \
  complexité O(log n)                          4
                                           hauteur = 3
                                           complexité O(n)
⚠ Arbre dégénéré — Pire cas
Si on insère des éléments dans l'ordre croissant dans un ABR (1, 2, 3, 4…), l'arbre dégénère en une liste chaînée. La hauteur devient n−1 et la recherche est en O(n). C'est pourquoi on utilise des arbres AVL ou arbres rouge-noir (hors programme) pour garantir l'équilibre.

Complexité spatiale

La récursivité consomme de la mémoire dans la pile d'appels. Pour un arbre de hauteur h :

  • Parcours DFS récursif : O(h) pile d'appels → O(n) au pire
  • Parcours BFS itératif : O(n) pour la file
07 — Points clés

Synthèse — Points clés à retenir

Définitions fondamentales

  • Arbre = graphe connexe acyclique orienté depuis une racine
  • Racine = unique nœud sans parent
  • Feuille = nœud sans enfant
  • Hauteur = longueur du plus long chemin racine→feuille
  • Arbre binaire : max 2 enfants par nœud
  • None représente l'arbre vide

Parcours à connaître

  • Préfixe : N G D (nœud en premier)
  • Infixe : G N D (nœud au milieu)
  • Postfixe : G D N (nœud en dernier)
  • Largeur : niveau par niveau, avec file
  • Tous en O(n)
  • Infixe sur ABR → ordre croissant

Fonctions récursives clés

  • taille(n) = 1 + t(g) + t(d)
  • hauteur(n) = 1 + max(h(g), h(d))
  • Cas de base : None → 0 ou -1
  • Recherche ABR : O(log n) moyen
  • Toujours tester le cas vide en premier

Pièges classiques

  • Oublier le cas de base (arbre vide)
  • Confondre hauteur et profondeur
  • ABR dégénéré si insertions triées
  • Hauteur arbre vide : -1 ou 0 selon convention
  • BFS : utiliser une file, PAS la récursion

Récapitulatif des fonctions essentielles
# ── TAILLE ─────────────────────────────────────────────
def taille(n):
    if n is None: return 0
    return 1 + taille(n.gauche) + taille(n.droite)

# ── HAUTEUR ────────────────────────────────────────────
def hauteur(n):
    if n is None: return -1
    return 1 + max(hauteur(n.gauche), hauteur(n.droite))

# ── PARCOURS PRÉFIXE ───────────────────────────────────
def prefixe(n):
    if n is None: return
    print(n.valeur); prefixe(n.gauche); prefixe(n.droite)

# ── PARCOURS INFIXE ────────────────────────────────────
def infixe(n):
    if n is None: return
    infixe(n.gauche); print(n.valeur); infixe(n.droite)

# ── PARCOURS POSTFIXE ──────────────────────────────────
def postfixe(n):
    if n is None: return
    postfixe(n.gauche); postfixe(n.droite); print(n.valeur)

# ── RECHERCHE ABR ──────────────────────────────────────
def recherche_abr(n, v):
    if n is None: return False
    if v == n.valeur: return True
    if v < n.valeur: return recherche_abr(n.gauche, v)
    return recherche_abr(n.droite, v)

🚨 Erreurs fréquentes au bac

  • ❌ Oublier le cas de base if noeud is None: return ...
  • ❌ Renvoyer la mauvaise valeur pour l'arbre vide (0 pour hauteur au lieu de -1)
  • ❌ Confondre None (arbre vide) et un nœud sans valeur
  • ❌ Croire qu'un ABR est toujours équilibré (il peut dégénérer)
  • ❌ Utiliser la récursion pour le parcours en largeur (il faut une file)
  • ❌ Dans le parcours infixe, écrire N→G→D au lieu de G→N→D

⭐ Niveau facile — Exercice 1

Lecture et manipulation d'un arbre

On considère l'arbre binaire suivant, construit en Python :

n4 = Noeud(4)
n7 = Noeud(7)
n2 = Noeud(2, n4, n7)
n9 = Noeud(9)
n5 = Noeud(5, n9, None)
r  = Noeud(6, n2, n5)

Questions :

  1. Dessinez l'arbre correspondant (schéma ASCII).
  2. Quelle est la hauteur de cet arbre ?
  3. Quelle est la taille (nombre de nœuds) ?
  4. Listez les feuilles de cet arbre.
  5. Donnez le résultat du parcours infixe.
▶ Indice

Pour dessiner l'arbre, repérez d'abord la racine r (valeur 6). Son fils gauche est n2 (valeur 2) et son fils droit est n5 (valeur 5). Continuez récursivement. La hauteur = nombre d'arêtes sur le chemin le plus long depuis la racine.

▶ Afficher la correction

1. Schéma de l'arbre :

       6
      / \
     2   5
    / \  /
   4   7 9

2. Hauteur : le chemin le plus long est 6→2→4 ou 6→2→7 ou 6→5→9, soit 2 arêtes → hauteur = 2.

3. Taille : on compte les nœuds : 6, 2, 5, 4, 7, 9 → 6 nœuds.

4. Feuilles : nœuds sans enfant : 4, 7, 9.

5. Parcours infixe (G→N→D) :

  • Sous-arbre gauche de 6 (racine=2) : 4 → 2 → 7
  • Nœud 6
  • Sous-arbre droit de 6 (racine=5) : 9 → 5

Résultat : 4 → 2 → 7 → 6 → 9 → 5

    if noeud.gauche is None and noeud.droite is None:
        return 1               # c'est une feuille
    return nb_feuilles(noeud.gauche) + nb_feuilles(noeud.droite)

# 2. Recherche d'une valeur (arbre quelconque)
def est_present(noeud, valeur):
    if noeud is None:
        return False
    if noeud.valeur == valeur:
        return True
    return est_present(noeud.gauche, valeur) or \
           est_present(noeud.droite, valeur)

# 3. Profondeur maximale (= hauteur)
def profondeur_max(noeud):
    if noeud is None:
        return -1              # convention : arbre vide → -1
    return 1 + max(profondeur_max(noeud.gauche),
                   profondeur_max(noeud.droite))

# 4. Parcours préfixe renvoyant une liste
def prefixe_liste(noeud):
    if noeud is None:
        return []               # liste vide pour l'arbre vide
    return ([noeud.valeur]
            + prefixe_liste(noeud.gauche)
            + prefixe_liste(noeud.droite))

# Question 5 — Classe Noeud pour arbre d'expression
class Noeud:
    def __init__(self, valeur, gauche=None, droite=None):
        self.valeur = valeur   # entier ou opérateur (str)
        self.gauche = gauche
        self.droite = droite

# Question 6 — Construction de l'arbre (3 + 5) × (8 − 2)
feuille_3 = Noeud(3)
feuille_5 = Noeud(5)
feuille_8 = Noeud(8)
feuille_2 = Noeud(2)
plus      = Noeud('+', feuille_3, feuille_5)
moins     = Noeud('-', feuille_8, feuille_2)
racine    = Noeud('*', plus, moins)

# Question 7 — Fonction d'évaluation récursive
def evaluer(noeud):
    """Évalue récursivement une expression arithmétique
    représentée par un arbre binaire d'expression."""
    if noeud is None:
        return 0

    # Cas de base : feuille = valeur numérique
    if noeud.gauche is None and noeud.droite is None:
        return noeud.valeur

    # Évaluation récursive des deux sous-expressions
    gauche_val = evaluer(noeud.gauche)
    droite_val = evaluer(noeud.droite)

    # Application de l'opérateur
    if noeud.valeur == '+':
        return gauche_val + droite_val
    elif noeud.valeur == '-':
        return gauche_val - droite_val
    elif noeud.valeur == '*':
        return gauche_val * droite_val
    elif noeud.valeur == '/':
        return gauche_val / droite_val

# Test
print(evaluer(racine))  # (3+5) × (8-2) = 8 × 6 = 48
🎓 Analyse
Cette fonction illustre parfaitement la puissance de la récursivité sur les arbres : on délègue le calcul des sous-expressions aux appels récursifs, et on combine simplement les résultats avec l'opérateur du nœud courant.
