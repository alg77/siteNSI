Architecture matérielle — Principaux composants

Périphériques

DÉFINITIONS
EXEMPLES
Périphérique
Dispositif connecté à une machine qui lui ajoute des fonctionnalités et lui permet d'interagir avec l'utilisateur.
Clavier, souris, écran, webcam, disque dur externe…
Périphérique d'entrée

Peut à la fois lire et écrire des informations.
Disque dur (HDD), SSD, clé USB, carte réseau, écran tactile.

Carte mère et bus

🔌 Carte mère
Circuit imprimé central qui assure la liaison entre tous les composants d'un ordinateur : processeur, mémoire vive, cartes d'extension et ports de connexion.
🚌 Bus
Un bus est un dispositif de transmission de données (câble ou piste électronique) qui relie les composants entre eux. On distingue le bus de données, le bus d'adresses et le bus de contrôle.
🃏 Cartes d'extension
Une carte d'extension (ou carte fille) permet d'ajouter des fonctionnalités à une machine. Elle est connectée à la carte mère via un bus. Exemples : carte graphique (GPU), carte réseau, carte son.

Les mémoires

La mémoire permet de stocker des données et des programmes. On distingue deux grands types :

  • Mémoire volatile : perd son contenu lors de la mise hors tension (ex. RAM).
  • Mémoire persistante : conserve son contenu sans alimentation électrique (ex. disque dur, ROM).

Unités de capacité

UnitéAbréviationÉquivalence
Kilo-octetKo10³ octets = 1 000 octets
Méga-octetMo10⁶ octets = 1 000 Ko
Giga-octetGo10⁹ octets = 1 000 Mo
Téra-octetTo10¹² octets = 1 000 Go
Péta-octetPo10¹⁵ octets = 1 000 To

Les 4 types de mémoire

1. Mémoire de masse (stockage)
Stockage long terme de grandes quantités de données. Technologies : HDD (électromécanique) ou SSD / clé USB (semi-conducteurs). Mémoire persistante.
Capacité : jusqu'à 10 To (HDD)
Vitesse : jusqu'à 500 Mo/s (SSD)
Volatile : NON
2. Mémoire vive (RAM)
Espace principal de stockage du microprocesseur en cours de fonctionnement. Appelée RAM (Random Access Memory) car le temps d'accès est faible et ne dépend pas de l'emplacement. Mémoire volatile.
Capacité : jusqu'à 32 Go
Vitesse : jusqu'à 2 Go/s
Volatile : OUI — perdue à l'extinction
3. Mémoire cache
Petites mémoires très rapides situées entre la RAM et le processeur. Évite le goulot d'étranglement en conservant les données fréquemment consultées. Très coûteuse, faible capacité. Hiérarchie : L1 (le plus rapide) → L2 → L3.
Capacité : quelques Ko (L1) à quelques Mo (L2)
Vitesse : jusqu'à 5 Go/s
Volatile : OUI
4. Mémoire morte (ROM)
Mémoire persistante dont le contenu est fixé à la fabrication. Contient les instructions de démarrage de la machine (firmware, BIOS/UEFI). Impossible à effacer par l'utilisateur. ROM = Read-Only Memory.
Capacité : quelques Mo
Volatile : NON
⚠ Piège classique
RAM ≠ disque dur. La RAM est rapide, volatile et de faible capacité (espace de travail). Le disque dur est lent, persistant et de grande capacité (stockage).
3

Le processeur (CPU)

🧠 Définition
Le processeur ou UCT (Unité Centrale de Traitement), en anglais CPU (Central Processing Unit), est le composant essentiel qui exécute les instructions machine des programmes informatiques.

Les 3 composants du CPU

ALU

Unité arithmétique et logique
Exécute tous les calculs arithmétiques (+, -, ×, /) et logiques (ET, OU, NON) du processeur.
⇄

Registres

Mémoire interne du CPU
Mémorisent en très petite quantité les données et instructions en cours de traitement. Mémoire la plus rapide de l'ordinateur.
⇄

Unité de contrôle

Chef d'orchestre du CPU
Séquence et coordonne les opérations : récupère les instructions et données en mémoire, les envoie à l'ALU, gère le résultat.
💡 À retenir
La fréquence d'horloge (en GHz) détermine le nombre d'opérations par seconde. Un CPU moderne possède plusieurs cœurs, permettant l'exécution parallèle de plusieurs tâches.

Architecture de Von Neumann

📐 Définition
L'architecture de Von Neumann (1945) est le modèle fondateur de l'ordinateur moderne : le programme et les données sont stockés dans la même mémoire. La quasi-totalité des ordinateurs actuels repose encore sur ce modèle.

L'unité de contrôle + l'ALU forment aujourd'hui le CPU.

🔑 Spécificité fondamentale
Dans le modèle de Von Neumann, les instructions sont traitées de la même façon que les données. C'est ce qui permet à un programme de modifier un autre programme.

Systèmes d'exploitation

💿 Définition
Un système d'exploitation ou OS (Operating System) est un ensemble de programmes qui gère les ressources matérielles et logicielles d'un ordinateur. Il sert d'intermédiaire entre les logiciels d'application et le matériel.

Fonctions d'un OS

Un système d'exploitation :

  • Exécute des programmes ;
  • Lit et écrit des informations dans les mémoires ;
  • Gère les fichiers et protège le système de fichiers ;
  • Gère les périphériques via des pilotes (drivers) ;
  • Communique entre ordinateurs via le réseau ;
  • Détecte et gère les erreurs matérielles et logicielles ;
  • Gère les droits et les permissions des utilisateurs.
🚀 Démarrage
Au démarrage, le firmware (stocké en ROM) s'exécute en premier, puis charge et lance le système d'exploitation depuis le disque dur.

Logiciel libre vs propriétaire

Logiciel LIBRE (free software)
Logiciel PROPRIÉTAIRE
L'utilisateur a la liberté d'exécuter, copier, distribuer, étudier, modifier et améliorer le logiciel. Son code source est accessible.
Le code source appartient à son éditeur. Il n'est ni accessible, ni distribuable, ni modifiable. L'utilisation est encadrée par une licence commerciale.
Exemples : Linux, Firefox, VLC, LibreOffice. Un logiciel libre n'est pas forcément gratuit.
Exemples : Windows, macOS, Adobe Photoshop. Un logiciel propriétaire n'est pas forcément payant (ex. Snapchat est gratuit mais propriétaire).

Familles de systèmes d'exploitation

🪟 Windows — Microsoft
  • Système propriétaire (code source fermé)
  • >70% de parts de marché sur PC (2021)
  • Utilisé sur PC, tablettes, consoles Xbox
  • Licence payante pour les utilisateurs
🐧 Unix / Linux
  • Système libre et open source
  • Descendants : macOS, iOS, Android, GNU/Linux
  • Utilisé dans les serveurs, smartphones, supercalculateurs
  • Code source accessible en ligne

Interface en ligne de commande (CLI)

💻 Définition
Une interface en ligne de commande est une interface textuelle entre l'utilisateur et la machine. L'utilisateur saisit des commandes au clavier, l'OS exécute et affiche les résultats. L'interpréteur de commandes s'appelle le Shell.

Syntaxe du prompt

Terminal — Shell bash
# Syntaxe du prompt pour un utilisateur standard :
nom_utilisateur@nom_machine:répertoire_courant$

# Exemple : Paul utilise l'ordinateur PC_Paul, répertoire courant = home
Paul@PC_Paul:home$

# Syntaxe du super-utilisateur (root = tous les droits) :
root@nom_machine:répertoire_courant#

Chemins d'accès

DÉFINITIONS
EXEMPLES (arbre ci-dessus)
Répertoire racine
Noté /, c'est le répertoire le plus haut dans l'arborescence. Il contient tous les autres répertoires.
Le répertoire / contient usr, etc, dev, tmp, bin, home…
Répertoire courant
Répertoire dans lequel on se trouve à un instant donné. Noté . (un seul point).
Si on a fait cd /home/Angie/NSI, le répertoire courant est NSI.
Répertoire père
Répertoire qui contient le répertoire courant. Noté .. (deux points).
Le répertoire père de NSI est Angie. cd .. depuis NSI ramène dans Angie.
Chemin absolu
Chemin qui part toujours de la racine /. Commence donc par /. Indépendant du répertoire courant.
/home/Angie/Danse/Photo
/home/Matéi/Exos/Exo1
Chemin relatif
Chemin qui part du répertoire courant. Ne commence pas par /. Le symbole .. permet de remonter d'un niveau.
Si courant = Angie : Danse/Photo
Si courant = NSI : ../Danse/Photo
Si courant = Angie → usr : ../../usr
📝 Exemple complet

Je suis dans /home/Angie/Jeux/Saisons et je veux accéder à /home/Matéi/Exos/Exo1 :

  • Chemin absolu : /home/Matéi/Exos/Exo1 (part toujours de /)
  • Chemin relatif : ../../../Matéi/Exos/Exo1 (remonte 3 niveaux : Saisons→Jeux→Angie→home→Matéi)

💡 Rappel : un chemin absolu commence par /, un chemin relatif ne commence jamais par /.

Droits et permissions d'accès

Utilisateurs et groupes

  • Chaque utilisateur est identifié par un identifiant (user id) et possède un répertoire personnel.
  • Le super-utilisateur (root) a accès à tous les fichiers et peut modifier tous les droits.
  • Le propriétaire d'un fichier est l'utilisateur qui l'a créé ; il peut modifier les droits de ses propres fichiers.
  • Chaque utilisateur appartient à un ou plusieurs groupes.

Lire les permissions avec ls -l

Sortie de la commande : ls -l

drwxr-xr--   2  Angie  enfants  4096  juin 26  Jeux
d = répertoire (- = fichier)  |   rwx r-x r-- = droits propriétaire | groupe | autres  |   Angie = propriétaire  |  enfants = groupe

Les 3 types de droits

LettreDe l'anglaisPour un fichierPour un répertoire
rread Lire le contenu du fichier Lister le contenu du répertoire
wwrite Modifier le contenu du fichier Créer, supprimer, renommer des fichiers dans le répertoire
xexecute Exécuter le fichier (s'il est exécutable) Traverser / ouvrir le répertoire
-— Droit absent (interdit)

Modifier les droits avec chmod

🔧 Syntaxe de chmod
chmod [qui][+/-/=][droits] nom_fichier
ParamètreSignificationValeurs possibles
QuiÀ qui s'applique la modificationu propriétaire · g groupe · o autres · a tous
TypeType de modification+ ajouter · - enlever · = assigner uniquement
DroitType d'autorisationr lecture · w écriture · x exécution
Exemples chmod
# Ajouter le droit d'exécution au propriétaire et au groupe
chmod ug+x nom_fichier

# Enlever le droit de lecture aux autres utilisateurs
chmod o-r nom_fichier

# Assigner rw pour le propriétaire, r pour le groupe, rien pour les autres
chmod u=rw,g=r,o= nom_fichier

# -R : récursif (applique aux sous-répertoires et fichiers contenus)
chmod -R g+rw Saisons

Notation octale

On peut aussi représenter les droits par un nombre en base 8 (trois chiffres : propriétaire, groupe, autres) :

DroitsBinaire (0=absent, 1=présent)Valeur octale
rwx1117
rw-1106
r-x1015
r--1004
-wx0113
-w-0102
--x0011
---0000
📐 Exemple
chmod 755 monfichier → propriétaire : rwx (7), groupe : r-x (5), autres : r-x (5)
chmod 644 monfichier → propriétaire : rw- (6), groupe : r-- (4), autres : r-- (4)

⭐⭐ Niveau 2 — Exercice 4
Lecture et interprétation des droits

La commande ls -l dans le répertoire Jeux d'Angie donne :

drwxr-x---  2  Angie  enfants  4096  Personnages
-rw-r--r--  1  Angie  enfants  1024  Mondes
drwxrwx---  2  Angie  enfants  4096  Saisons

Angie et Matéi appartiennent au groupe enfants. Leurs parents ont leur propre groupe parents (= « autres utilisateurs »).

  1. Pour Personnages : complétez le tableau (oui/non) pour Angie, Matéi, et un parent.
  2. Pour le fichier Mondes : que peuvent faire les parents ?
  3. Pour Saisons : Matéi peut-il créer un fichier dedans ? Justifiez.
  4. Quelle commande donne à tous les utilisateurs le droit de lire Saisons ?
  5. Exprimer les droits de Saisons en notation octale.
▶ Voir la correction

1. Personnages (drwxr-x---) :

LectureÉcritureExécution/Traverser
Angie (propriétaire)✅ oui✅ oui✅ oui
Matéi (groupe enfants)✅ oui❌ non✅ oui
Parent (autres)❌ non❌ non❌ non

2. Mondes (-rw-r--r--) : Les parents peuvent lire le fichier Mondes (droit r présent pour « autres »). Ils ne peuvent pas le modifier ni l'exécuter.

3. Saisons (drwxrwx---) : Oui, Matéi peut créer un fichier dans Saisons. Le droit w du groupe (rwx) lui permet de créer/supprimer des fichiers dans ce répertoire.

4. chmod a+r Saisons ou chmod o+r Saisons (ajouter le droit r aux autres).

5. drwxrwx--- → propriétaire : rwx=7, groupe : rwx=7, autres : ---=0 → 770

⭐⭐⭐ Niveau 3 — Exercice 5
Problème complet — Organisation d'un projet NSI

Paul (paul@PC_NSI) veut organiser ses projets NSI. Son répertoire personnel est /home/paul.

  1. Donnez la structure complète de l'arborescence à créer, puis les commandes pour créer la structure suivante depuis /home/paul :
paul/
├── NSI/
│   ├── Python/
│   │   ├── TP1/
│   │   └── TP2/
│   └── HTML/
└── Perso/
  1. Paul est dans /home/paul/NSI/Python/TP1. Il veut copier tous les fichiers .py dans TP2. Quelle commande utiliser ?
  2. Paul crée un fichier secret.py dans TP1. Il veut que lui seul puisse le lire et le modifier. Quelle commande utiliser ?
  3. Il veut que TP2 soit accessible en lecture/exécution par son groupe, et inaccessible aux autres. Les droits actuels sont rwxr-xr-x. Quelle commande minimale utiliser ?
  4. Exprimer les droits finaux de TP2 (après la question 4) en notation octale.
▶ Indice

Pour la question 2, pensez au caractère joker *. Pour les droits, calculez bit par bit : r=4, w=2, x=1.

▶ Voir la correction

1. Commandes :

mkdir NSI
mkdir NSI/Python
mkdir NSI/Python/TP1
mkdir NSI/Python/TP2
mkdir NSI/HTML
mkdir Perso

2. Depuis /home/paul/NSI/Python/TP1 :
cp *.py ../TP2/
Le * remplace n'importe quelle chaîne, donc *.py sélectionne tous les fichiers .py.

3. chmod 600 secret.py → rw------- : seul le propriétaire peut lire (r) et écrire (w), personne d'autre ne peut rien faire.

4. Droits actuels : rwxr-xr-x. On veut supprimer le droit des autres (o) :
chmod o-rx TP2 (ou chmod o= TP2)
Résultat : rwxr-x---

5. rwxr-x--- → propriétaire : rwx=7, groupe : r-x=5, autres : ---=0 → 750
